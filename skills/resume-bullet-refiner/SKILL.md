# Resume Bullet Refiner

## Overview

Transform weak bullets into strong, specific, impact-first statements using Google's XYZ formula. No job description needed.

**Formula:** "Accomplished [X] as measured by [Y], by doing [Z]."
- X = result or outcome
- Y = metric (%, $, volume, time, scale)
- Z = specific action or method

## Rules

1. **Action verb first.** Never: "responsible for," "helped with," "assisted in," "worked on," "involved in."

2. **Name the technology.** Vague: "built a pipeline." Strong: "built an AWS Glue pipeline."

3. **One to two lines max per bullet.**

4. **Cut filler:** "various," "multiple," "different," "successfully," "effectively," "utilized."

5. **Buzzword ban:** "spearheaded," "passionate," "results-driven," "detail-oriented," "leveraged" (unless technical), "synergy." Replace with plain verb + result.

6. **No first-person pronouns.** Never "I," "my," "me."

7. **Signal soft skills through context**, not labels. Never write "demonstrated leadership." Instead: "Led 3-engineer squad to deliver..."

## Metrics Handling

When a bullet has no metric, do NOT fabricate or estimate. Instead:

**Step 1 — Ask with specific options.** Based on the bullet's context, suggest 2–3 concrete things the user can check:

> "This bullet could be quantified. Do you have any of these?
> - Pipeline runtime before vs. after your change (seconds/minutes)?
> - Number of records processed per run?
> - Error rate reduction or SLA improvement %?"

**Step 2 — If user has no metrics:** Rewrite using scale/scope/comparative language:
- Scale: "across 12 microservices"
- Scope: "used by 200+ internal engineers"
- Comparative: "replacing a 4-hour manual process"
- Complexity: "handling 3 concurrent data sources with schema drift"

## Action Verb Upgrade Table

| Weak | Strong alternatives |
|------|---------------------|
| Managed | Led, Owned, Directed |
| Worked on | Built, Engineered, Developed |
| Helped | Contributed, Enabled, Accelerated |
| Made | Designed, Architected, Implemented |
| Used | Deployed, Integrated, Automated |
| Updated | Migrated, Refactored, Optimized |

## Output Format

For every bullet:
1. **Original:** [bullet as-is]
2. **Refined:** [rewritten bullet]
3. **Why stronger:** [one line]
4. **Always end with:** "Apply this to the resume?" — wait for explicit confirmation before writing any file.

If metrics are missing, show Step 1 question first. Wait for user reply. Then rewrite.

> **CRITICAL — NEVER SKIP:** Do NOT write to any file until the user confirms the rewrite. Show the refined bullet, then stop. Apply only after explicit approval ("yes", "apply", "looks good", etc.). Auto-writing without confirmation is a violation of this skill.

## Common Mistakes

| Mistake | Fix |
|---------|-----|
| "Leveraged X to improve Y" | "Reduced Y by N% using X" |
| Two actions crammed in one bullet | Split into two bullets |
| Outcome buried at end | Lead with outcome, end with method |
| Generic tech ("cloud tools") | Specific tech ("AWS Step Functions + Lambda") |
| "Successfully deployed" | "Deployed" — success is implied |
| Passive voice ("was built by") | Active: "Built..." |
